[BvS Encryption & Decryption Tool]

Project Announcement
We will be adapting the decryption tool from the Batman Vs Superman movie with the ability to encrypt data.
Posted on 11 June 2019 BvS Encryption & Decryption Tool: Project Announcement
Posted on 11 June 2019

Encryption is the process of encoding data in way where only authorised users can view. Decryption is the process of making encoded data readable again.

In Batman Vs Superman, Batman steals encrypted data from Luthor's private server. In order to view this data he has to run a decryption program. We will be creating an application with the same look and feel as the one in BvS. This application will have the ability to encrypt and decrypt files.

FUIs (Fake User Interfaces), including this one, are only designed to look good. This means that the layout or text styling are usually not very user-friendly. For example, in this FUI the text is in an incredibly small font for which to read you have to move closer. We can resolve this issue by removing the unneccesary information from the screen and using a larger font size.

We should start by analysing the UI in the different stages of the decryption scene. The first stage is where Batman inserted the drive into the computer to which a CONNECTED message was displayed.

The middle container contains a drive letter and what appears to be its status. The drive letter is highly likely referring to the drive containing the stolen encrypted data. There appears to be a command line interface (CLI) beneath it. A CLI is a text-based UI used to interact with the OS and applications. Typed into the CLI, there is writing that seems to say crypyt following an unreadable word and a filepath. The crypt most likely refers to a command-line encryption utility that is used to decrypt the files. The rest of the information surrounding the middle container is illegible.

In the second stage the user is told that its INITIATING DECRYPTION.

The third stage is the decryption stage. This stage allows us to see the right-hand pane. On the right-hand pane, there is a list of SkyServers that Batman could interact with. SkyServer provides astronomers and scientific institutions with SDSS's research. They have made detailed 3-D maps of the Universe. Since this isn't related to encryting/decrypting data we can ignore it.

The middle container contains information on the decryption status, a drive letter and a file path. /dev is the location of device files in a Linux Filesystem Hierarchy. With this information the file path /dev/LEECH_01 may refer to the data copied from Luthor's server.

As decryption progresses, the percent number increments and the red progress bar expands.

The fourth and final stage is when the decompression is complete. Once decryption is complete the middle container expands to the left and right-hand side.

Based on this breakdown we will write a specification on what the project should have. We then need to draw a map of the different screens and their layout.
